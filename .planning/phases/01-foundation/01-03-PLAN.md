1. **src/lib/auth/claims.ts** — Server-only role management functions:
   - Add `import 'server-only'` at top
   - `setUserRole(uid: string, roleLevel: RoleLevel, promotedByUid: string)`:
     a. Call `getAdminAuth().setCustomUserClaims(uid, { roleLevel })`
     b. Also update `users/{uid}` document in Firestore with `{ roleLevel, updatedAt: serverTimestamp() }` — denormalized for queries that filter by role
     c. Log to `roleAuditLog` collection: `{ targetUid: uid, newRole: roleLevel, promotedBy: promotedByUid, timestamp: serverTimestamp() }`
     d. Return success
   - `getUserRoleLevel(uid: string): Promise<RoleLevel>`:
     a. Call `getAdminAuth().getUser(uid)`
     b. Return `customClaims?.roleLevel ?? ROLE_LEVELS.guest`
   - `searchUsersByEmail(emailPrefix: string)`:
     a. Use Firestore `users` collection query where email starts with prefix (for admin search)
     b. Return array of `{ uid, email, displayName, roleLevel }`

2. **src/app/(main)/admin/actions.ts** — Server Actions for admin page:
   - `'use server'` directive
   - `promoteUser(formData: { targetUid: string, newRole: RoleLevel })`:
     a. Get current user's tokens via `getTokens(await cookies(), {...})`
     b. Verify caller's roleLevel: super admin (4) can set any role; admin (3) can only set up to moderator (2); anything else rejects
     c. Verify new role is not higher than caller's role (no privilege escalation)
     d. Call `setUserRole(targetUid, newRole, callerUid)`
     e. Return `{ success: true }` or `{ error: string }`
   - `searchUsers(query: string)`:
     a. Verify caller is at least admin (roleLevel >= 3)
     b. Search by email prefix in Firestore users collection
     c. Return matching users with their current roles

3. **firestore.rules** — Complete security rules:
   - `rules_version = '2'`
   - Helper functions: `roleLevel()`, `isRegistered()`, `isModerator()`, `isAdmin()`, `isSuperAdmin()`, `isOwner(uid)`
   - `/users/{uid}`: read if authenticated; write if owner OR admin+
   - `/roleAuditLog/{entry}`: read if admin+; write: false (Admin SDK only, bypasses rules)
   - Add comments explaining defense-in-depth: these rules are the REAL security layer, not middleware
   - Keep rules extensible — future phases will add collections here

4. **scripts/seed-super-admin.ts** — One-time super admin setup:
   - Standalone script using firebase-admin directly (not Next.js)
   - Reads `SUPER_ADMIN_EMAIL` from environment
   - Looks up user by email, sets `{ roleLevel: 4 }` custom claims
   - Also creates/updates the `users/{uid}` Firestore document with `{ roleLevel: 4, email }` for consistency
   - Console output confirming success
   - Add npm script in package.json: `"seed:admin": "npx tsx scripts/seed-super-admin.ts"`
   - Add `SUPER_ADMIN_EMAIL` to `.env.example`
