4. **src/app/(auth)/register/page.tsx** — Registration page ("use client"):
   - Centered Card component on navy background
   - Heading: "Create Your Account" in Cinzel
   - Form using react-hook-form with zodResolver(registerSchema)
   - Fields: Email (Input component), Password (Input with type="password"), Confirm Password
   - Submit button: Button component with gold variant, loading state during submission
   - On submit:
     a. Call `createUserWithEmailAndPassword(auth, email, password)` from firebase/auth
     b. Get the ID token: `await user.getIdToken()`
     c. POST to `/api/login` with the ID token in the body (creates session cookie)
     d. Call a Server Action `registerUser(uid)` that sets `roleLevel: 1` via Admin SDK `setCustomUserClaims`
     e. Force token refresh: `await user.getIdToken(true)` to pick up new roleLevel claim
     f. POST to `/api/login` again with refreshed token (updates cookie with roleLevel)
     g. Redirect to `/dashboard` via `router.push('/dashboard')`
   - Error handling: show Firebase error messages (email already in use, weak password) as inline form errors
   - Link to login page: "Already have an account? Sign In"

   - Fields: Email, Password
   - On submit:
     a. Call `signInWithEmailAndPassword(auth, email, password)`
     b. Get ID token, POST to `/api/login` (creates session cookie)
     c. Redirect to `/dashboard`
