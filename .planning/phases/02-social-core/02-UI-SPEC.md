## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline pill gaps |
| sm | 8px | Gap between avatar and handle, chip internal padding |
| md | 16px | Default element spacing, post card internal padding |
| lg | 24px | Section padding, compose box vertical rhythm |
| xl | 32px | Profile header vertical padding |
| 2xl | 48px | Major section breaks (feed top margin below category tabs) |
| 3xl | 64px | Page-level top padding (below fixed 70px navbar) |

Exceptions:
- Navbar height is 70px (pre-existing, from Phase 1 `--spacing-nav`)
- Avatar touch targets: minimum 44px × 44px on mobile (accessibility minimum)
- Circular avatar on profile header: 80px × 80px (desktop), 64px × 64px (mobile)
- Notification bell touch target: 44px × 44px

## Typography

All sizes are in px. Two weights only: regular (400) for body/reading, semibold (600) for labels/headings. Cinzel is uppercase by default via tracking-widest utility.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | EB Garamond | 16px | 400 | 1.5 | Post text, bio, comment text, link preview body |
| Label | Cinzel | 12px | 600 | 1.2 | Category chip text, jurisdiction badge, nav links, button text, input labels, @handle |
| Heading | Cinzel | 20px | 600 | 1.2 | Section titles, post detail page heading, profile display name |
| Display | Cinzel | 28px | 600 | 1.1 | Profile page name (large), page-level headings |

Supporting scale notes:
- Subtext / metadata (timestamp, like count, follower count): EB Garamond 14px, weight 400, color `text-mid` (#b8a888)
- Error messages: EB Garamond 14px italic, color `crimson` (#8b1a1a)
- "Edited" timestamp on edited posts/comments: EB Garamond 12px italic, color `text-mid`

## Color

The Byzantine palette is locked from Phase 1. No new color values may be introduced in Phase 2.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `bg-navy` | #0d1b2e | Page background, feed background |
| Secondary (30%) | `bg-navy-mid` | #152338 | Post cards, profile header card, compose box, dropdown menus |
| Tertiary surface | `bg-navy-light` | #1e3352 | Input fields, comment input, hover states on cards |
| Accent (10%) | `text-gold` / `bg-gold` | #c9a84c | See reserved list below |
| Accent bright | `text-gold-bright` | #e8c96a | Hover state on gold elements |
| Accent dim | `text-gold-dim` | #8a6f30 | Input borders, secondary gold text, input labels |
| Body text | `text-text-light` | #e8dfc8 | Primary readable text (post body, bio, comments) |
| Subtext | `text-text-mid` | #b8a888 | Timestamps, metadata, follower counts, placeholder text |
| Destructive | `text-crimson` / `bg-crimson` | #8b1a1a | Delete post, delete comment, block action — destructive only |

**Accent (#c9a84c) is reserved for:**
- Follow button (filled gold variant)
- Post submit button ("Share to Agora")
- Like button when in liked/active state (gold fill)
- Jurisdiction badge border and text on profile page
- Active category filter tab underline/indicator
- Profile display name text on profile page
- Notification bell with unread count badge background
- Verified checkmark icon on post cards and profile

**Accent is NOT used for:** general body text, card borders (use `border-gold/[0.15]`), comment timestamps, follower counts, or any passive display elements.

**Card border rule:** All card borders use `border-gold/[0.15]` (15% opacity gold) matching the existing Card component. No solid gold borders on cards.

**Default banner fallback:** If a user has not set a profile banner, render a repeating Byzantine tile pattern using the existing gold grid overlay at 8% opacity on `bg-navy-mid` — this matches the `body::before` pattern from globals.css at higher visibility.

## Component Inventory

New components to build in Phase 2. All extend existing tokens and follow Phase 1 patterns.

### PostCard
- Base: `Card` component (navy-mid bg, gold/15% border, 6px rounded)
- Layout: avatar (36px circle) + handle + display name + jurisdiction badge + timestamp in one row; post text below; optional image below text; category chip + like/comment row at bottom
- Jurisdiction badge: `font-cinzel text-xs` gold-dim text, displayed as `• Antiochian` inline after handle
- Verified checkmark: gold SVG check icon (16px), shown when `roleLevel >= 2`; positioned inline after display name
- Category chip: pill with `bg-navy-light border border-gold/[0.15] font-cinzel text-[10px] uppercase tracking-widest text-gold-dim px-2 py-0.5 rounded-full`
- Like button: heart icon — idle state uses `text-text-mid`; liked state uses `text-gold` with fill; optimistic update on click
- Comment count: speech-bubble icon + count as a `<Link>` to `/agora/[postId]`; `text-text-mid`; hover: `text-gold`
- Three-dot overflow (own posts only): opens dropdown with Edit and Delete options

### ProfileHeader
- Banner area: 200px height (desktop), 140px height (mobile); full-width; bg `navy-mid` with tile fallback or user-uploaded image
- Avatar: 80px circle (desktop) / 64px circle (mobile); absolute-positioned overlapping banner bottom; `border-2 border-gold/40`
- Display name: Cinzel 28px gold, below avatar
- @handle: Cinzel 14px text-mid
- Jurisdiction badge: Cinzel 12px, gold-dim text, displayed as a full badge pill with border; communicates canonical trust
- Bio: EB Garamond 16px text-light, max 2 lines on profile card with expand option
- Stats row: "N Posts · N Followers · N Following" — Cinzel 12px uppercase, text-mid for label, text-light for number
- Follow button: gold variant Button (md size) — "Follow" / "Following" (outline variant when already following)
- Message button: outline variant Button (md size), disabled state (`opacity-50 cursor-not-allowed`), tooltip on hover: "Direct messaging coming soon"
- Three-dot overflow: Block, Mute, Report User (listed in that order; report de-emphasized, no icon prominence)

### ProfileTabs
- Two tabs: "Posts" and "Media"
- Active tab: Cinzel 12px uppercase, gold text, 2px gold underline border-bottom
- Inactive tab: Cinzel 12px uppercase, text-mid, hover: text-light

### ComposeBox
- Container: Card component as base
- Textarea: extends Input component styling, min-height 80px, auto-grows with content; placeholder: "Share in the Agora..."
- Photo attach button: outline icon button (paperclip/image icon), 44px touch target
- Category row: AI-suggested chip shown after text input pauses; "Change" link opens manual picker sheet
- Category picker: bottom sheet / popover with 10 Orthodox categories as selectable chips
- Submit button: gold Button ("Share to Agora"), disabled when textarea empty

### CategoryFilterTabs
- Horizontal scrollable row, no wrap, scrollbar hidden
- Pills: "All" + 10 category names — Cinzel 11px uppercase, tracking-widest
- Active: gold text, 2px gold underline; inactive: text-mid; hover: text-light
- Row padding: 16px horizontal, 8px vertical; border-bottom: `border-gold/[0.10]`

### NotificationBell
- Bell SVG icon (24px), `text-text-light`, hover `text-gold`
- Unread badge: 16px circle, `bg-gold text-navy font-cinzel text-[10px]`, absolute top-right of bell
- Positioned in Navbar right section, before avatar

### CommentCard
- Simpler than PostCard — no image, no category chip, no jurisdiction badge in card body
- Avatar (28px circle) + @handle + timestamp inline
- Comment text: EB Garamond 16px text-light
- "Edited" label: EB Garamond 12px italic text-mid, shown after timestamp when edited
- Delete/Edit buttons (own comments only): text-mid, hover text-crimson for delete

### LinkPreviewCard
- Embedded within PostCard when post body contains a URL
- Container: `bg-navy-light border border-gold/[0.10] rounded-md overflow-hidden`
- Layout: thumbnail image (if available) on left or top; title (Cinzel 14px text-light); description (EB Garamond 13px text-mid, 2-line clamp); domain label (Cinzel 11px text-mid)

### JurisdictionDropdown
- Two sections separated by a `<hr>` in `border-gold/[0.15]`
- Section 1 label: "Canonical Eastern Orthodox Churches" (Cinzel 10px uppercase text-gold-dim)
- Section 2 label: "Other Christians" (Cinzel 10px uppercase text-gold-dim)
- Each option: EB Garamond 16px text-light, hover bg-navy-light

### BlockingSkeletons
- PostCard skeleton: grey shimmer blocks matching PostCard layout proportions
- Use `animate-pulse bg-navy-light` for all skeleton blocks
- Show 3 skeleton cards on initial feed load

## Interaction Contracts

### Likes (Optimistic Update)
1. User taps like — heart immediately fills gold, count increments by 1 in UI
2. Firestore write fires in background
3. On success: no UI change (already correct)
4. On failure: heart returns to empty, count decrements, toast error shown

### Follow / Unfollow
1. Follow button switches to "Following" (outline variant) immediately on tap
2. Firestore `follows` collection write fires in background
3. On failure: revert to "Follow" (gold variant), show toast: "Could not follow. Try again."

### Feed Pagination
- Infinite scroll using Firestore cursor pagination (10 posts per batch)
- Spinner (gold animate-spin) shown at bottom of feed while loading next batch
- No "load more" button — scroll-triggered only

### Comment Submission
- Textarea clears immediately on submit (optimistic)
- Comment appears at bottom of flat thread immediately
- On failure: textarea repopulated with draft, toast error shown

### Post Deletion Flow
1. Three-dot menu → "Delete Post"
2. Confirmation dialog (modal): "Delete this post? This cannot be undone." with "Delete" (crimson Button) and "Keep" (outline Button)
3. On confirm: post removed from feed immediately (optimistic); Firestore delete fires
4. On Firestore failure: post reappears, toast error shown

### Comment Deletion Flow
Same pattern as post deletion. "Delete" button color: crimson. "Keep" button: outline.

### Block Flow
1. Three-dot overflow on profile → "Block"
2. Confirmation dialog: copy as specified above
3. On confirm: navigate away from profile to feed; blocked user's content filtered client-side

### Category AI Classification
1. User types post text; after 800ms debounce, Firebase AI / Gemini call fires
2. Suggested category chip appears below textarea with "Change" link
3. If user clicks "Change": category picker sheet opens (10 categories as grid of selectable chips)
4. If AI confidence low or text < 20 chars: skip suggestion, show "What best describes this post?" prompt with category picker directly
5. Selected category stored with post on submit

## Accessibility Requirements

- All interactive elements have visible focus ring: `focus-visible:ring-2 focus-visible:ring-gold/60` (matching Button component)
- Avatar images require descriptive `alt` text: "{displayName}'s avatar"
- Like button `aria-label`: "Like post" / "Unlike post" (toggled)
- Notification bell `aria-label`: "Notifications, {N} unread" (or "Notifications, no unread")
- Comment count link `aria-label`: "View {N} comments"
- Three-dot overflow button `aria-label`: "Post options" / "Profile options"
- Category filter tabs: `role="tablist"` with each tab as `role="tab"` and `aria-selected`
- Confirmation dialogs: `role="dialog"` with `aria-modal="true"` and focus trap
- Skeleton loaders: `aria-busy="true"` on feed container while loading; `aria-label="Loading posts"`
- Jurisdiction dropdown: `aria-label="Select jurisdiction"` with grouped options using `<optgroup>` or listbox role
