## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tag gaps |
| sm | 8px | Compact element spacing, avatar-to-text gap |
| md | 16px | Default element spacing, card internal gaps |
| lg | 24px | Section padding, between-card gaps |
| xl | 32px | Page section breaks, calendar day header padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Touch targets for prev/next day arrows: minimum 44px (accessibility — WCAG 2.5.5)
- Expandable saint card content area: no minimum height — collapses to content
- Calendar toggle pill: 36px height (compact inline control, not primary action)

Source: Established in Phase 1 (`src/styles/globals.css` `--spacing-nav: 70px` as precedent for named exceptions); 8-point scale confirmed in prior phase components.

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 16px | 400 (regular) | 1.5 | EB Garamond | Saint life summaries, fasting rule text, member card details |
| Label | 12px | 400 (regular) | 1.4 | Cinzel uppercase tracking-widest | Calendar toggle label, section headers (FEASTS, SAINTS, FASTING), jurisdiction badge text, filter tab labels |
| Heading | 20px | 600 (semibold) | 1.2 | Cinzel | Saint card name, Synodeia page title, calendar section title |
| Display | 28px | 600 (semibold) | 1.2 | Cinzel | Calendar day date header (e.g. "19 March 2026") |

Notes:
- Cinzel is an all-caps / small-caps typeface — all Cinzel usage is inherently uppercase
- EB Garamond italic is reserved for error messages (matches existing Input.tsx `italic` pattern) and Synaxarion excerpt previews
- Scripture reading references (disabled stubs) use EB Garamond 14px / weight 400 / opacity 50% — consistent with existing `text-text-mid/50` pattern in codebase

Source: `src/components/ui/Button.tsx` (font-cinzel, tracking-widest), `src/components/ui/Input.tsx` (font-garamond, text-base=16px, italic errors), `src/components/agora/CategoryFilterTabs.tsx` (text-xs=12px, font-cinzel), `04-RESEARCH.md` (feast level treatment recommendations).

## Color

All tokens defined in `src/styles/globals.css` @theme block. No new colors introduced in Phase 4.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `bg-navy` | #0d1b2e | Page background, body |
| Secondary (30%) | `bg-navy-mid` | #152338 | Cards (Card.tsx), calendar day view container, member cards |
| Surface tertiary | `bg-navy-light` | #1e3352 | Skeleton shimmer fill, input backgrounds, calendar toggle inactive track |
| Accent (10%) | `text-gold` / `border-gold` | #c9a84c | See accent reserved list below |
| Accent dim | `text-gold-dim` | #8a6f30 | Input labels, active filter underline, feast level 1-2 text |
| Accent bright | `text-gold-bright` | #e8c96a | Gold button gradient highlight, Great Feast text |
| Accent pale | `text-gold-pale` | #f0dfa0 | Great Feast card background tint (feast_level >= 5) |
| Text primary | `text-text-light` | #e8dfc8 | All body text, member display names |
| Text secondary | `text-text-mid` | #b8a888 | Saint card collapsed excerpt, jurisdiction badge, city/state location text |
| Destructive | `text-crimson` / `border-crimson` | #8b1a1a | Destructive actions only (none in Phase 4 primary flows; reserved for future toggle-off confirmation if needed) |

**Accent reserved for:**
1. Active filter tab — bottom border `border-gold` + text `text-gold`
2. Calendar toggle active state — border `border-gold` + text `text-gold`
3. Jurisdiction badge outline — `border-gold/[0.30]` border
4. Section dividers — `border-gold/[0.10]` (matches existing pattern)
5. Great Feast cards — `border-gold` full-opacity border + `text-gold-bright` title
6. Expand/collapse chevron icon on saint cards — `text-gold`
7. Prev/next day navigation arrows — `text-gold`
8. "Coming soon" tooltip accent — `text-gold-dim` label

**Feast level color mapping** (from orthocal.info `feast_level` field):
- `feast_level >= 5` (Great Feast): `text-gold-bright` title + `border-gold` card border + `bg-gold-pale/[0.05]` card tint
- `feast_level 3-4` (Polyeleos / Vigil): `text-gold` title + `border-gold/[0.30]` card border
- `feast_level 1-2` (Minor Feast): `text-gold-dim` title + `border-gold/[0.15]` card border (standard Card.tsx)
- `feast_level 0` (Liturgy / ordinary): `text-text-light` — no highlight

Source: `src/styles/globals.css` (all tokens), `src/components/ui/Card.tsx` (border-gold/[0.15] baseline), `04-RESEARCH.md` (feast level treatment in Claude's Discretion section).

## Interaction Contracts

### Calendar Day View

| Interaction | Behavior |
|-------------|----------|
| Prev day arrow | Navigates to `?date=YYYY-MM-DD` (previous day); Server Component re-fetches orthocal.info |
| Next day arrow | Navigates to `?date=YYYY-MM-DD` (next day); Server Component re-fetches |
| Calendar toggle (Old Julian / New Julian) | For logged-in users: calls `updateCalendarPreference` Server Action; persists to `userProfiles.calendarPreference`; page re-renders with new calendar endpoint. For guests: client-side state only, defaults to New Julian |
| Saint card expand | Click/tap expands inline — full Synaxarion HTML rendered below the 2-3 sentence excerpt; chevron rotates 180deg; no modal, no separate page |
| Saint card collapse | Click/tap collapses back to excerpt; chevron returns to 0deg |
| Reading reference (disabled stub) | Non-interactive `<span>` with `cursor-not-allowed` and `title="Scripture Library — coming soon"` tooltip; no click handler |

### Synodeia People Finder

| Interaction | Behavior |
|-------------|----------|
| Jurisdiction filter tab | Filters member grid to selected jurisdiction; "All" tab shows all canonical Orthodox members; tab selection is mutually exclusive with active search (selecting a tab clears search input, entering search clears tab to "All") |
| Name search input | Debounced 300ms; calls Server Action with prefix keyword; results update member grid |
| Member card | Non-interactive display card — no click action in Phase 4; avatar, display name, jurisdiction badge, city/state (conditionally shown) |
| Location sharing toggle (profile edit page) | Toggle on/off; calls `updateLocationSharing` Server Action; when toggled off, city/state input collapses and data is hidden from other users immediately |

### Loading States

| State | Component | Pattern |
|-------|-----------|---------|
| Calendar day loading | `CalendarSkeleton` | `aria-busy="true"` div; 3 skeleton rows for feast/saint/reading sections; `bg-navy-light animate-pulse rounded` (matches BlockingSkeletons.tsx pattern) |
| Synodeia grid loading | `SynodiaSkeleton` | `aria-busy="true"` div; 6 skeleton member card shapes in responsive grid; avatar circle + 2 text bars |
| Calendar toggle saving | Button `loading` prop | Gold button shows spinner while `updateCalendarPreference` Server Action is in flight |

## Accessibility Contracts

| Element | Requirement |
|---------|-------------|
| Calendar toggle | `role="group"` + `aria-label="Calendar system"`, each option is a `<button>` with `aria-pressed` state |
| Prev/next day arrows | `aria-label="Previous Day"` / `aria-label="Next Day"` — icons are `aria-hidden="true"` |
| Saint card expand | `aria-expanded` on the trigger button; `aria-controls` pointing to the collapsible content `id` |
| Jurisdiction filter tabs | `role="tablist"` + `role="tab"` + `aria-selected` — matches existing CategoryFilterTabs pattern |
| Skeleton states | `aria-busy="true"` + `aria-label="Loading {content}"` on wrapper |
| Reading ref disabled | `aria-disabled="true"` on the `<span>`; `title` attribute for tooltip |
| Member card location | City/state wrapped in `<span aria-label="{city}, {stateRegion}">` for screen reader clarity |
| Feast level Great Feast | `aria-label="Great Feast: {feast title}"` on the feast title element |
