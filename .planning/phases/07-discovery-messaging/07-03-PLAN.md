  <action>
    Present the user with the verification checklist below. Start the dev server with `npx next dev` if not already running. Wait for user to complete each verification step and report results.

    **Search (SRCH-01, SRCH-02):**
    1. Open the app on desktop. Verify the search input appears in the navbar (center-right area).
    2. Type a keyword (e.g., "grace") in the navbar search bar. Verify it navigates to /search?q=grace.
    3. On the /search page, verify tabs: All | Videos | Posts | People | Scripture | Church Fathers.
    4. Verify the "All" tab shows up to 5 results per content type section with section headers.
    5. Click "See all Scripture results" — verify it switches to the Scripture tab.
    6. On an individual tab, verify "Load More" button appears if results exceed 10.
    7. Clear the search input — verify empty state: "Search Ekklesia Agora" heading.
    8. Search for a nonsense term — verify "No results for '[query]'" message.
    9. On mobile (or narrow viewport), verify the magnifying glass icon appears instead of the text input. Tap it to go to /search.

    **Messaging (MSG-01, MSG-02, MSG-03):**
    10. Navigate to any user's profile page. Verify a "Send Message" button appears (not on your own profile).
    11. Click "Send Message" — verify it navigates to /messages and opens (or creates) a conversation.
    12. In the message thread, type a message and send. Verify it appears immediately (right-aligned, gold-tinted bubble).
    13. Open a second browser/incognito as the other user. Verify the message appears in real-time without refresh (left-aligned, navy bubble).
    14. Verify the envelope icon in the navbar shows an unread badge for the recipient.
    15. Have the recipient open the conversation. Verify: (a) the unread badge clears, (b) "Seen" indicator appears on the sender's last message.
    16. Navigate to /messages. Verify the conversation list shows: avatar, display name, last message preview, timestamp, and unread dot for unread conversations.
    17. Verify online presence: if both users are active, green dot appears on the avatar in the conversation list.
    18. On mobile viewport, verify: conversation list is the default /messages view; tapping a conversation navigates to /messages/[id] (full-page thread with back button).
