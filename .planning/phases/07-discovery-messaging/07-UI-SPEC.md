## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding |
| sm | 8px | Compact element spacing (search result row gap, message timestamp gap) |
| md | 16px | Default element spacing (card padding, input padding, message bubble padding) |
| lg | 24px | Section padding (search page section headers, conversation list item padding) |
| xl | 32px | Layout gaps (search page content area top padding) |
| 2xl | 48px | Major section breaks (empty state vertical centering offset) |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Navbar icon touch targets: 44px (w-11 h-11) — established pattern from NotificationBell.tsx, must be maintained for search icon, envelope icon, and send button
- Conversation list sidebar: 280px min-width, 320px max-width — fixed, not 8-point, matches CONTEXT.md locked decision
- Message input area: minimum 44px height for touch accessibility

Source: NotificationBell.tsx line 146 (w-11 h-11 pattern), CONTEXT.md §Specific Ideas

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0d1b2e` (`bg-navy`) | Page background, message thread background, search page background |
| Secondary (30%) | `#152338` (`bg-navy-mid`) | Conversation list sidebar, search result cards, message composer bar, dropdowns |
| Accent (10%) | `#c9a84c` (`text-gold` / `bg-gold`) | See accent reserved list below |
| Destructive | `#8b1a1a` (`text-crimson`) | Sign Out in avatar dropdown (already established — no new destructive actions in Phase 7) |

Accent (`#c9a84c`) reserved for:
1. Unread badge on envelope icon (bg-gold text-navy) — mirrors NotificationBell badge exactly
2. Unread dot on conversation list items (bg-gold, w-2 h-2 rounded-full)
3. Online presence green dot is NOT gold — use `bg-green-500` (the only non-Byzantine color in Phase 7; justified by universal convention for "online")
4. Type badge text on search result cards (text-gold font-cinzel font-bold)
5. "See all [type] results" link text on the All tab (text-gold hover:text-gold-bright)
6. Send button hover state (hover:text-gold)
7. Search input focus ring (focus-visible:ring-gold/60 — matches existing Input.tsx pattern)
8. Section header underline accent on active search tab (border-b-2 border-gold)

Secondary semantic:
- `#1e3352` (`bg-navy-light`) — active/hovered conversation row, unread message row background (matches NotificationBell.tsx line 191 bg-navy-light/50 for unread items)
- `rgba(201,168,76,0.15)` (`border-gold/[0.15]`) — all card borders, sidebar separator (matches Card.tsx)
- `#b8a888` (`text-text-mid`) — secondary text: timestamps, context lines, message previews in conversation list
- `#e8dfc8` (`text-text-light`) — primary text: display names, message body text

Source: globals.css @theme, Card.tsx border-gold/[0.15], NotificationBell.tsx badge and unread patterns

## Interaction Contracts

### Search Flow

| Trigger | Behavior |
|---------|----------|
| User types in navbar search input (desktop) | 300ms debounce, then router.push('/search?q={query}') |
| User taps search icon (mobile) | Navigates to /search (empty state shown) |
| User types in /search page input | 300ms debounce, then router.push('/search?q={query}&tab={currentTab}') |
| Tab click | router.push('/search?q={query}&tab={tabId}') — no client state |
| "See all Videos results" link | Switches to Videos tab: router.push('/search?q={query}&tab=videos') |
| "Load more" button (individual tab) | Client-side: appends 10 more results from pre-fetched data (Server Component fetches 50 per type; client renders first 10 then adds 10 per click) |

### Messaging Flow

| Trigger | Behavior |
|---------|----------|
| Click envelope icon in navbar | navigate('/messages') |
| Click "Message" button on /profile/[handle] | navigate('/messages?to={uid}') — Server Action createOrGetConversation runs, then redirects to /messages/{conversationId} |
| Click conversation in list (desktop) | Renders MessageThread in right panel (no page navigation) |
| Click conversation in list (mobile) | navigate('/messages/{conversationId}') — full-page thread |
| Send message | Server Action sendMessage writes to conversations/{id}/messages; onSnapshot delivers to both participants; scrolls to bottom |
| Recipient opens conversation | Server Action markMessagesRead resets unreadCounts[uid] to 0 on conversation doc; updates seenAt/seenBy on last received message |
| Online presence | lastSeen field updated via fire-and-forget Server Action on page load + window focus event |

### Loading States (Claude's Discretion — resolved here)

| Surface | Skeleton |
|---------|----------|
| Search results loading | 5 skeleton rows per section: bg-navy-mid rounded animate-pulse. Title line: w-48 h-3 bg-navy-light. Context line: w-32 h-2 bg-navy-light mt-1. |
| Conversation list loading | 3 skeleton rows: avatar circle w-10 h-10 bg-navy-light animate-pulse, name line w-24 h-3 bg-navy-light, preview line w-40 h-2 bg-navy-light mt-1. |
| Message thread loading | 4 alternating skeleton bubbles: right-aligned (w-48 h-8 bg-gold/10) and left-aligned (w-40 h-8 bg-navy-mid) with animate-pulse. |

## Layout Contract

### /search Page

```
<main class="max-w-3xl mx-auto px-4 pt-[86px]">  ← 70px nav + 16px gap
  <section>                                         ← search input (full width)
  <nav>                                             ← tabs (All | Videos | Posts | People | Scripture | Fathers)
  <div>                                             ← results (sections on All, list on type tabs)
</main>
```

- Max content width: 768px (max-w-3xl), centered
- Tab bar: font-cinzel text-xs uppercase tracking-widest, active tab: text-gold border-b-2 border-gold, inactive: text-text-mid hover:text-text-light
- Section headers on All tab: font-cinzel text-xl text-gold uppercase tracking-widest, border-b border-gold/[0.15] pb-2 mb-4

```
<div class="flex h-screen pt-[70px]">
```
