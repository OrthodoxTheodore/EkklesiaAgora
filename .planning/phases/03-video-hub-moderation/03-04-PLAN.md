**2. Create `src/app/(main)/videos/[id]/page.tsx`** as Server Component:

Per UI-SPEC /videos/[id] layout (YouTube-style):

**Auth and data loading:**
- Get tokens from cookies (user may or may not be authenticated)
- Fetch video by ID from Firestore (`videos/{id}`)
- If video not found or (status !== 'published' AND viewer is not uploader AND viewer is not moderator+): show notFound()
- Call incrementViewCount(videoId) Server Action on every page load
- Fetch video comments: query `videos/{id}/comments` ordered by createdAt asc
- Check if current user has liked: read `videos/{id}/likes/{uid}`
- Fetch related videos: query `videos` where `channelId == video.channelId AND status == 'published' AND videoId != current`, limit 5. If fewer than 5, backfill with `videos` where `category == video.category AND status == 'published'`, limit (5 - channelResults.length)
