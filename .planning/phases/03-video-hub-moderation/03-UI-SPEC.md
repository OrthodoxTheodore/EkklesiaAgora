## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip internal padding, inline gaps |
| sm | 8px | Compact element spacing, button gaps |
| md | 16px | Default element spacing, card internal gaps |
| lg | 24px | Section padding, card-to-card gap |
| xl | 32px | Layout column gaps, page section breaks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:

| Exception | Value | Reason |
|-----------|-------|--------|
| Video player aspect ratio container | 16:9 fixed ratio | Not a spacing token — CSS aspect-ratio property |
| Duration chip overlay on thumbnail | px-2 (8px) horizontal, py-1 (4px) vertical | Multiples of 4; compact overlay chip |
| Upload progress bar height | 8px (sm) | Matches browser native range thumb expectation |
| Moderation review card player | No horizontal padding on player itself | Full-width player by design |
| Touch targets — video controls | 44px minimum tap area | Accessibility requirement, WCAG 2.5.5 |

Source: established in Phase 1 (globals.css, PostCard.tsx patterns). No new font sizes introduced.

## Component Inventory

New components for this phase (reuse/extend existing patterns):

### VideoCard
Extends PostCard pattern. Grid card used in feeds, channel pages, /videos browse.
- Thumbnail at top (16:9 aspect ratio, rounded-t-[6px])
- Duration chip: bottom-right overlay on thumbnail — `bg-black/70 font-cinzel text-xs text-white px-2 py-1 rounded`
- Below thumbnail: channel avatar (w-9 h-9, rounded-full, border-gold/[0.15]) + video title (font-cinzel text-base text-text-light) + metadata row (view count, relative date, category chip)
- No description excerpt in card view (locked decision from CONTEXT.md)
- Inline playback: clicking thumbnail starts HTML5 video in-card; title/metadata remain as link to /videos/[id]

### VideoCard skeleton
Mirrors BlockingSkeletons pattern. Thumbnail rectangle (aspect-video, bg-navy-light animate-pulse) + 3 text bar rows.

### ChannelCard
Used on /channels browse page.
- Channel logo/avatar (w-12 h-12, rounded-full, border-gold/[0.15])
- Channel name (font-cinzel text-base text-text-light)
- Primary category chip (font-cinzel text-xs text-gold-dim, bg-navy-light border-gold/[0.15], rounded-full)
- Subscriber count (font-cinzel text-xs text-text-mid)

### VideoPlayer
HTML5 `<video>` element with `controls` attribute. Responsive: `w-full aspect-video rounded-[6px]`. No custom control skin — native browser controls. Fullscreen via browser native. Aria-label="Video player".

### UploadProgressBar
During Firebase Storage resumable upload:
- Container: `w-full bg-navy-light rounded-full h-2`
- Fill: `bg-gold h-2 rounded-full transition-all` — width driven by percentage state
- Label above: `font-cinzel text-xs text-text-mid` showing "Uploading… 42%"
- On completion: label changes to "Processing…" with a pulse indicator

### ModerationReviewCard (pending uploads tab)
- Full-width VideoPlayer at top
- Below player: title, description, tags, thumbnail preview, category (font-garamond text-base text-text-light)
- Uploader context block (bg-navy-light border border-gold/[0.15] rounded-[6px] p-4 mt-4):
  - Avatar + display name + handle
  - Jurisdiction badge (existing pattern from PostCard)
  - Account age, post history count, verification status (font-cinzel text-xs text-text-mid)
- Action row (mt-6 flex gap-3 justify-end):
  - "Approve" — Button gold variant, size sm
  - "Request Changes" — Button outline variant, size sm (opens freeform note textarea)
  - "Reject" — Button gold variant with crimson override (existing pattern from delete confirmation in PostCard)

### FlaggedContentCard (flagged content tab)
- Same as ModerationReviewCard
- Additional context block: flag reason(s), flagger count (font-cinzel text-xs text-text-mid)

### ChannelApplicationForm
Fields: Channel name (Input), Channel type (personal / institutional — native select with existing JurisdictionDropdown pattern), Description (textarea, font-garamond text-base, bg-navy-mid border-gold/[0.15] rounded-[6px] p-3), Primary Orthodox category (native select).
Submit: Button gold variant "Submit Application".

## Interaction States

### Upload Flow States
| State | Visual |
|-------|--------|
| Idle (pre-upload) | Form visible, "Upload Video" CTA gold button |
| Uploading | Progress bar visible, button disabled with spinner, "Uploading… N%" label |
| Processing (upload complete, Firestore write pending) | Progress bar at 100%, label "Processing…" with pulse |
| Pending review | Status badge: `bg-navy-light border border-gold/[0.15] font-cinzel text-xs text-gold-dim px-2 py-1 rounded-full` — "Pending Review" |
| Approved | Status badge: gold text "Published" |
| Rejected | Status badge: crimson text "Rejected" + reason shown |
| Changes requested | Status badge: text-text-mid "Changes Requested" + moderator note shown |

### Video Player States
| State | Visual |
|-------|--------|
| Loading | Thumbnail shown until video can play (native browser behavior) |
| Error | `font-garamond text-base text-crimson` — "Video unavailable. Try refreshing the page." below player area |
| Playing | Native controls |
| Inline playback | Player replaces thumbnail within card; clicking outside navigates to /videos/[id] |

### Moderation Console States
| State | Visual |
|-------|--------|
| Queue empty (pending) | Empty state (see Copywriting) |
| Queue empty (flagged) | Empty state (see Copywriting) |
| Action pending | Approve/Reject/Request Changes button shows loading spinner (Button loading prop) |
| Request changes — note input | Textarea expands below action row; "Send Note" Button gold sm |

### Channel Page States
| State | Visual |
|-------|--------|
| No videos | Empty state (see Copywriting) |
| Loading videos | VideoCard skeletons (3 cards, same animate-pulse pattern as BlockingSkeletons) |

## Page Layout Contracts

### /videos (Video Browse / Hub Home)
- Layout: same main content width as /agora
- CategoryFilterTabs at top (reuse/adapt existing component)
- Video grid: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6`

### /videos/[id] (Video Detail Page)
- Layout: two-column on desktop (`lg:grid lg:grid-cols-3 gap-6`), single column on mobile
- Left/main col (lg:col-span-2): VideoPlayer → metadata block → like/share/flag/category → comments
- Right sidebar (lg:col-span-1): "More from this channel" heading + vertical VideoCard list (3–5 cards); below that "More in [category]" if same-channel list is short
- Sidebar collapses to below comments on mobile

### /channel/[handle] (Channel Page)
- Banner: full-width, 200px height, bg-navy-light (if no banner uploaded)
- Header: channel avatar (w-20 h-20, -mt-10 relative, border-4 border-navy, rounded-full) overlapping banner
- Channel name (font-cinzel text-2xl), subscriber count (font-cinzel text-xs text-text-mid), description (font-garamond text-base)
- Subscribe button (Button gold md)
- Video grid below: same 3-column grid as /videos

### /channels (Channel Browse)
- CategoryFilterTabs (adapt from CategoryFilterTabs.tsx)
- Channel grid: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6`
- Each cell: ChannelCard

### /admin/moderation (Moderation Console)
- Inherits existing /admin layout and role guard
- Two tabs: "Pending Uploads" | "Flagged Content" — same tab styling as CategoryFilterTabs
- Each tab: vertical stack of ModerationReviewCard / FlaggedContentCard with `gap-6`
