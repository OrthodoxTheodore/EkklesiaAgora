## Critical Gaps (Blockers)

### GAP-1: AGRA-05 — Moderator cannot delete posts

**Severity:** Critical
**Requirement:** AGRA-05 — Moderator can delete any post/comment
**Phase:** 02-social-core

`src/app/actions/posts.ts` line 144 guards `deletePost` with author-only check:
- Guard: `postData.authorUid !== uid` — no moderator bypass before this check
- A moderator calling `deletePost` on another user's post always gets an authorization error
- Contrast: `deleteVideo` in `videos.ts` correctly adds `callerRoleLevel < 2` as a secondary authorization path

**Fix:** Add `callerRoleLevel >= 2` moderator bypass before the author-only guard.

---

### GAP-2: CAL-05/06/07 — Calendar reading links may produce broken Scripture URLs

**Severity:** High
**Requirements:** CAL-05, CAL-06, CAL-07
**Phase:** 04-orthodox-identity

`extractReadingRefs` in `orthocal.ts` extracts book name from `reading.display.split(/[\s.]/)[0]`, producing abbreviations like `"Heb"` or `"1Cor"`. `ReadingRef.tsx` line 11 looks these up in `BOOK_ABBREV_MAP`, which uses full names as keys (`"Hebrews"`, `"1 Corinthians"`). Mismatches produce undefined, causing broken URLs like `/scripture/heb/9`.

**Fix:** Add abbreviation-to-full-name normalization in `extractReadingRefs` or `ReadingRef.tsx`.

---
